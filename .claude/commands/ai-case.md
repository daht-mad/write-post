# AI 활용 사례 게시글 작성

DEVLOG.md를 기반으로 비개발자 대상 AI 활용 사례 게시글을 작성합니다.

## 사용법

```
/ai-case
```

## 프로세스

### 1단계: DEVLOG 분석
- `DEVLOG.md` 파일을 읽고 전체 작업 흐름 파악
- 주요 작업, 사용한 도구, 결과물 정리
- 사용자에게 파악한 내용 요약해서 확인받기

### 2단계: 맥락 질문 (한 번에 하나씩)
다음 항목들에 대해 **한 번에 하나씩** 질문하며 충분한 맥락 수집:

1. **타겟 독자**: 이 글을 읽을 사람들은 누구인가요? (직군, 상황 등)
2. **문제 상황 (Before)**: AI 사용 전에 어떤 불편함/문제가 있었나요? 구체적인 상황과 소요 시간은?
3. **시작 계기**: 이 작업을 **지금** 시작하게 된 구체적인 계기가 있나요? (왜 더 이상 미룰 수 없었는지)
4. **AI 선택 이유**: 왜 이 AI 도구를 선택했나요? 다른 도구와 비교 고민이 있었나요?
5. **작업 과정 하이라이트**: DEVLOG에 기록된 작업들 중 **특히 강조하고 싶은 작업**이 있나요? (게시글에 포함할 핵심 작업 선정)
6. **인상적인 순간**: AI와 협업하면서 "오!" 했던 순간이 있나요?
7. **막혔던 부분**: AI가 잘 못 이해하거나 막혔던 부분은? 어떻게 해결했나요?
8. **결과 임팩트**: 결과물로 인해 구체적으로 뭐가 달라졌나요? (시간 절약, 효율 등)
9. **팀 협업 변화**: 이 결과물이 팀과 협업하는 데 어떤 도움이 되었나요?
10. **핵심 팁**: 다른 사람에게 꼭 알려주고 싶은 AI 활용 팁이 있나요?
11. **향후 계획**: 이 결과물을 더 발전시킬 계획이 있나요?

**중요: 질문은 반드시 한 번에 하나씩만!**
- 메인 질문이든 꼬리질문이든 한 메시지에 질문 하나만
- 여러 질문을 한꺼번에 던지지 않기

**추천 답변 제공**: 각 질문 시 DEVLOG를 기반으로 가장 적절한 추천 답변도 함께 제시
- 추천 답변은 3개 내외로 bullet point 형식으로 제공
- 사용자가 "응", "그거로 해" 등으로 간단히 수락 가능하게
- 추천 답변이 맞지 않으면 사용자가 수정/보완
- **추천 답변 제시 후 항상 "더 추천해드릴까요?" 옵션 제공** - 사용자가 원하면 추가 추천

**꼬리질문**: 답변이 모호하거나 더 구체적인 내용이 필요하면 추가 질문 (역시 한 번에 하나씩)
**향후 계획 제안**: 사용자가 계획이 없다고 하면, DEVLOG와 맥락을 바탕으로 "이런 것도 가능할 것 같아요"라고 2-3가지 제안해서 시야를 넓혀주기
**충분한 맥락 확보 시**: 다음 단계로 진행

### 3단계: 제목 선정
초안 작성 직전, **후킹 가능한 제목 3-4개를 추천**하고 사용자에게 선택받기:

**제목 추천 기준:**
- 비개발자가 클릭하고 싶은 제목
- 구체적인 임팩트 포함 (시간, 숫자 등)
- 결과물이나 도구를 명확하게
- "~했다", "~만들었다" 같은 완료형 표현

**예시:**
> 제목 추천드려요. 어떤 게 좋을까요?
> 1. "비개발자가 Claude Code로 오전 2시간 만에 매출 대시보드를 만들었다"
> 2. "스프레드시트 노가다 끝! AI로 3,500개 데이터 자동 분류한 후기"
> 3. "개발자 없이 Airtable 대시보드 만든 비개발자의 AI 협업기"
> 4. (직접 입력하기)

사용자가 선택하거나 수정 요청하면 반영

### 4단계: 초안 작성
아래 템플릿에 맞춰 `AI_CASE_STUDY.md` 파일로 작성

### 5단계: 피드백 및 수정
- 초안 작성 후 사용자에게 수정할 부분 질문
- 피드백 반영하여 최종 수정

## 작성 시 주의사항

1. **코드 제외**: 비개발자 대상이므로 코드 블록 사용 금지
2. **프롬프트는 포함**: 실제 사용한 자연어 프롬프트는 인용문(`>`)으로 포함
3. **스크린샷 권장**: 결과물, Before/After는 시각적으로 보여주기
4. **구체적 숫자**: "많이 절약" 대신 "주 2시간 절약" 같이 구체적으로
5. **실패 사례도 포함**: 막혔던 부분과 해결 과정이 오히려 더 도움됨
6. **복붙 가능한 프롬프트**: 독자가 바로 사용할 수 있게 템플릿화
7. **스토리텔링으로 작성**: 딱딱한 "상황-요청-결과" 나열이 아닌, 자연스럽게 읽히는 이야기처럼
8. **작업 과정은 DEVLOG 기반**: DEVLOG의 실제 작업 순서와 맥락을 살려서 에피소드처럼 구성
9. **Claude 작업 내용은 구체적으로**: "코드 작성" 대신 "SDK 설치하고, API 라우트 만들고, 데이터 불러오는 로직까지" 처럼 서술형으로
10. **헤딩2(##)에는 이모지 필수**: 각 섹션 제목 앞에 적절한 이모지 추가 (📝, 🎯, 😫, 💡, 🛠️, 🔧, ✅, 💬, 🌍, 🚀, 📋 등)

## 5단계: 마무리 안내

### 이미지 추천
글 작성이 완료되면, 독자 이해를 돕기 위해 **최소 3개 이미지**를 추천:

1. **어떤 섹션에** 어떤 이미지가 들어가면 좋을지 (Before/After, 작업 화면, 결과물 등)
2. 섹션 이름으로 안내 (코드 라인 번호 X)

예시:
> **추천 이미지:**
> - "문제 상황" 섹션: 기존 방식의 불편함을 보여주는 스크린샷
> - "AI와 협업한 과정" 섹션: Claude와 대화하며 작업하는 터미널 화면
> - "결과" 섹션: 완성된 결과물 전체 화면

### 게시판 업로드 안내
```
📋 게시판에 올리는 방법:

1. AI_CASE_STUDY.md 파일에서 마우스 우클릭 → "미리보기 열기" (또는 Cmd+Shift+V)
2. 미리보기 화면에서 Cmd+A로 전체 선택
3. 게시판 작성 화면에 Cmd+V로 붙여넣기
4. 추천된 위치에 이미지 추가
5. 게시!
```
